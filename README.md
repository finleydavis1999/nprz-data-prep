# nprz-app

Analytics app for Dutch CBS microdata (PC4 + gemeente). SvelteKit 5 + MapLibre GL + DuckDB-WASM + Protomaps.

## Stack

- **Frontend**: SvelteKit (Svelte 5 runes), Tailwind v4 (token-only via `@theme`), MapLibre GL with Protomaps "white" basemap.
- **Data layer**: DuckDB-WASM in a worker, parquet files OPFS-cached and registered as `BROWSER_FSACCESS` file handles. Versioned cache invalidation keyed on `manifest.version`.
- **Cartography**: `simple-statistics` (Jenks via `ckmeans`, quantile, equal-interval), `chroma-js` palettes, MapLibre `step` paint expressions, `svelteplot` histogram.
- **Auth**: `better-auth` with `better-sqlite3` (no ORM), email/password, sessions in `local.db`. No public sign-up; users are seeded from `seed-users.json` via `npm run users:seed`. Dev can bypass auth with `AUTH_DISABLED=true` in `.env`.
- **R pipeline**: `duckdb`, `sf`, `rmapshaper`, `geojsonio`. Source SQLite + shapefile/gpkg → parquet + GeoJSON + TopoJSON.

## Run

Prereqs: Node 22+, R 4.4+ with `duckdb`, `sf`, `rmapshaper`, `geojsonio`, `dplyr`, `jsonlite`, `DBI`. Source data in `raw-data/` (gitignored).

```sh
cp .env.example .env                       # set BETTER_AUTH_SECRET (32+ chars), AUTH_DISABLED=true for dev
npm install
npm run data                               # one-time: build static/data/{parquet,geo,manifest.json}
cp seed-users.json.example seed-users.json # edit, then:
npm run users:seed                         # idempotent — only needed when AUTH_DISABLED is unset
npm run dev
```

Open <http://localhost:5173>. With `AUTH_DISABLED=true` in `.env` the app loads directly; without it, sign in at `/login` using a seeded account.

## Layout

```
R/                         # data pipeline (npm run data → Rscript R/build.R)
src/
  lib/
    cartography/           # classify, palettes, step expression, Legend, Histogram
    data/                  # manifest, duckdb, opfs-cache, query
    map/                   # Map, ChoroplethLayer, BoundaryLayer, basemap, context
    server/                # auth (better-auth on better-sqlite3)
    state/                 # selection, cartography, overlay (singleton $state)
    ui/                    # Field, Panel, ScaleToggle, *Picker, *Filters, *Controls
  routes/
    +layout.server.js      # session guard → /login (skipped when AUTH_DISABLED=true)
    +page.svelte / .server # main map view + logout action
    login/                 # email/password sign-in (no public sign-up)
static/data/               # generated by `npm run data` (gitignored)
tests/e2e/                 # Playwright specs
raw-data/                  # source SQLite + geo (gitignored)
```

## Conventions

- **Svelte 5 runes**: `$state` field declarations on classes for singleton state; `$derived.by(...)` for non-trivial derivations; no `export let` / `$:` / `<script context="module">`.
- **Map context**: child map layers consume the parent map via `getMapContext()` (Symbol-keyed, exported from `src/lib/map/context.js`); never `getContext('map')`.
- **Field rows**: any "label + control" row uses `<Field label="...">` to keep label-column widths and density consistent.
- **Tokens only**: every `<style>` block consumes CSS custom properties (`var(--text-sm)` etc.) defined in `src/routes/layout.css` `@theme`. No raw color/font-size literals.
- **Imports for browser-only deps** (maplibre-gl, pmtiles, etc.) are dynamic inside `onMount` so SSR doesn't choke.
