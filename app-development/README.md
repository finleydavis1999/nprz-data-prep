# CBS Map App — Interactive Spatial Data Visualization

A modern SvelteKit web application for exploring Dutch CBS (Statistics Netherlands) spatial data interactively. Uses **DuckDB-WASM** for client-side SQL queries and **MapLibre GL** for dynamic choropleth mapping.

## Quick Start

### Prerequisites
- **Node.js** ≥ 18
- **Parquet data files** (generated by R pipeline)

### Setup

1. **Generate data** (if not already done):
   ```bash
   cd R_scripts/app_core_setup
   # Run 01_setup_and_process.R in R
   ```

2. **Copy data to static folder:**
   ```bash
   cp ../first_data/export/*.parquet ./static/data/
   ```

3. **Install dependencies:**
   ```bash
   npm install
   ```

4. **Start development server:**
   ```bash
   npm run dev
   ```

5. **Open in browser:**
   Navigate to `http://localhost:5173`

## Features

### Interactive Variable Selection
- **34+ demographic & housing variables** exposed via dropdown
- Instantly switch between population, age groups, household types, dwelling characteristics, etc.
- Real-time map updates with no data reload

### Dynamic Choropleth Mapping
- **Quantile-based classification:** 4-class scheme calculated from data distribution
- **Automatic data cleaning:** Removes suppressed (-99995, -99997) and negative values
- **Color-coded legend:** Updates with class breaks for each variable
- **"No data" category:** Grey color for features with missing/invalid data

### Client-Side Processing
- **DuckDB-WASM:** Queries run entirely in the browser — no server needed
- **Parquet format:** Efficient compression, fast load times
- **No backend required:** Deploy as static files to any web host

### Map Controls
- **Zoom, pan, tilt:** Full MapLibre GL interactivity
- **Layer switching:** View different administrative scales (grid, buurt, wijk, gemeente, PC4)
- **Responsive design:** Works on desktop and tablet

## Architecture

### Component Structure

```
src/
├── routes/
│   └── +page.svelte          # Main map & dashboard (all logic here)
├── lib/
│   ├── index.ts              # Utilities (optional)
│   └── assets/               # Images, icons
└── app.html                  # Root HTML template
```

### Data Flow

```
+page.svelte (SvelteKit page component)
    ├── Initialize DuckDB-WASM in browser
    ├── Load parquet files as virtual tables
    ├── Listen to variable dropdown changes
    ├── Execute SQL query to calculate quantile breaks
    ├── Generate GeoJSON with class assignments
    ├── Pass GeoJSON to MapLibre GL
    └── Render choropleth with legend
```

### +page.svelte Logic

#### 1. **Variable Configuration**
```typescript
const variables = [
    { key: 'aantal_inwoners', label: 'Total population' },
    // ... 33 more variables
];
```

#### 2. **DuckDB Initialization**
- Instantiates WASM module on page load
- Registers each parquet file as a queryable table

#### 3. **Quantile Calculation**
For each variable selected:
```sql
SELECT 
  QUANTILE(column_name, [0, 0.25, 0.5, 0.75, 1.0]) as breaks
FROM table_name
WHERE column_name NOT IN (-99995, -99997) 
  AND column_name > 0;
```

#### 4. **Class Assignment**
Uses breaks to assign each feature to one of 4 quantile classes

#### 5. **GeoJSON Rendering**
Converts DuckDB results to GeoJSON → MapLibre GL → choropleth

## Files Explained

### Static Data (`static/data/`)

Each GeoParquet file from the R pipeline is available as a layer:

| File | Layer Name | Features |
|------|-----------|----------|
| `grid_100m_rijnmond.parquet` | grid_100m | ~6,000 cells |
| `buurt_2024.parquet` | buurt | ~300 neighbourhoods |
| `wijk_2024.parquet` | wijk | ~3,500 wards |
| `gemeente_2024.parquet` | gemeente | ~344 municipalities |
| `pc4_zh_2024.parquet` | pc4 | ~1,500 postcodes |

**Note:** Only one layer renders at a time (select via dropdown)

### Styling

- **Colors:** Light blue (low) → Dark blue (high) quantile colors
- **Borders:** Light grey with 0.5px stroke
- **Opacity:** 0.8 (allows subtle viewing of overlapping geometry)
- **Customizable:** Edit the `FillLayer` color scheme in `+page.svelte`

## Extending the App

### Add a New Variable

Edit `+page.svelte` to add to the `variables` array:

```typescript
const variables = [
    { key: 'aantal_inwoners', label: 'Total population' },
    // Add this:
    { key: 'my_new_column', label: 'My new variable label' },
];
```

The dropdown and query logic will automatically adapt.

### Switch to a Different Data Layer

Modify the `current_layer` variable:

```typescript
let current_layer = 'wijk'; // Change to 'gemeente', 'pc4', 'buurt', etc.
```

### Change Color Scheme

Edit the `FillLayer` in `+page.svelte`:

```svelte
<FillLayer
    id="choropleth"
    paint={{
        'fill-color': [
            'step',
            ['feature-state', 'class'],
            '#e8e8e8', // no-data grey
            0, '#e1f5ff', // q0 light blue
            1, '#81d4fa', // q1 medium blue
            2, '#0288d1', // q2 darker blue
            3, '#01579b'  // q3 darkest blue
        ],
        'fill-opacity': 0.8
    }}
/>
```

### Add Multiple Layer Toggle

Wrap layer names in state and add radio buttons:

```typescript
let selectedLayer = $state('grid_100m');

const layers = [
    { id: 'grid_100m', label: '100m Grid' },
    { id: 'buurt', label: 'Neighbourhoods' },
    // ...
];
```

Then in template:
```svelte
{#each layers as layer}
    <label>
        <input type="radio" bind:group={selectedLayer} value={layer.id} />
        {layer.label}
    </label>
{/each}
```

## Troubleshooting

### Map doesn't load
- **Check:** Parquet files exist in `static/data/` with correct names
- **Check:** Browser console for DuckDB errors (F12 → Console)
- **Check:** Node version ≥ 18 (`node --version`)

### Variables dropdown is empty
- **Cause:** DuckDB initialization failed
- **Fix:** Check console for WASM load errors
- **Fix:** Verify parquet file format is valid (run R pipeline)

### Map shows no data
- **Common:** Column name doesn't exist in parquet file
- **Check:** Export/verify column names from R output
- **Check:** Ensure column names match exactly (case-sensitive)

### Development server won't start
```bash
# Clear build cache and reinstall
rm -rf node_modules .svelte-kit
npm install
npm run dev
```

### Data looks wrong (weird colors/values)
- **Check:** Did you re-run the R pipeline? 
- **Check:** Are suppressed values (-99995, -99997) being filtered correctly?
- **Fix:** Verify DuckDB query in console:
  ```sql
  SELECT COUNT(*), MIN(column_name), MAX(column_name) 
  FROM layer_name
  WHERE column_name > 0 AND column_name NOT IN (-99995, -99997);
  ```

## Production Deployment

### Build for production

```bash
npm run build
```

Generates a static site in `build/` ready to deploy.

### Deploy to Vercel, Netlify, GitHub Pages, etc.

Since this is a static SvelteKit app, it works anywhere:

```bash
# Vercel
vercel deploy ./build

# Or copy build folder contents to any static host
```

### Performance Notes

- **Bundle size:** ~500 KB (DuckDB-WASM + MapLibre GL + dependencies)
- **Data load:** Parquet files stream in as needed (lazy loading)
- **Query time:** <100ms for quantile calculation (browser-dependent)
- **Rendering:** ~500ms for large layer (geometry rasterization)

